# STM32 Example

## Overview

This guide shows how to use `esp_serial_flasher` from an STM32 MCU to flash an Espressif SoC over UART. Because STM32 projects are chip-specific and generated by STM32CubeMX, no ready-made project is provided here. Follow the steps below to set one up for your MCU.

You will need two UART instances and two GPIO outputs:

- One UART for communication with the ESP target (flashing).
- One UART for debug log output (optional).
- Two GPIO outputs to control the ESP target's `BOOT` and `RESET` pins.

For wiring details see the [Hardware Connections Guide](../../docs/hardware-connections.md#uartserial-interface).

## Step 1 — Configure Peripherals in STM32CubeMX

1. Open STM32CubeMX and create a new project for your MCU.
2. Enable:
   - A **UART** for communication with the ESP SoC (TX, RX). Set baud rate to 115200, 8N1, no flow control.
   - A **second UART** for debug output (optional).
   - **Two GPIO outputs** for `BOOT` and `RESET` control of the ESP SoC. Set their initial output level to High.
3. In the **Project Manager** tab set *Toolchain / IDE* to **CMake**.
4. Click **Generate Code**.

## Step 2 — Integrate esp-serial-flasher into CMakeLists.txt

Open the top-level `CMakeLists.txt` generated by CubeMX and add the following **after** `add_subdirectory(cmake/stm32cubemx)`:

```cmake
set(FLASHER_DIR /path/to/esp-serial-flasher)   # adjust to your checkout

set(PORT STM32)
add_subdirectory(${FLASHER_DIR} ${CMAKE_BINARY_DIR}/flasher_lib)
```

Then link the library to your executable:

```cmake
target_link_libraries(${CMAKE_PROJECT_NAME}
    stm32cubemx
    flasher
)
```

## Step 3 — Application Code

The code below shows the key parts to add inside `Core/Src/main.c`. For a complete working example including the `flash_binary()` helper and full flashing loop, refer to any of the other examples in this repository (e.g. `examples/esp32_example`).

### Includes (`USER CODE BEGIN Includes`)

```c
#include "stm32_port.h"
#include "esp_loader.h"
```

### Port initialisation and connect (`USER CODE BEGIN 2`, inside `main()`)

```c
esp_loader_t loader;

stm32_port_t port = {
    .port.ops     = &stm32_uart_ops,
    .huart        = &huart2,           // UART instance connected to the ESP SoC
    .port_boot    = TARGET_BOOT_GPIO_Port,
    .pin_num_boot = TARGET_BOOT_Pin,
    .port_rst     = TARGET_RESET_GPIO_Port,
    .pin_num_rst  = TARGET_RESET_Pin,
};

esp_loader_init_uart(&loader, &port.port);

esp_loader_connect_args_t connect_cfg = ESP_LOADER_CONNECT_DEFAULT();
if (esp_loader_connect(&loader, &connect_cfg) != ESP_LOADER_SUCCESS) {
    Error_Handler();
}
```

> [!NOTE]
> `TARGET_BOOT_GPIO_Port`, `TARGET_BOOT_Pin`, etc. are generated by CubeMX when you assign user labels to the GPIO pins. If you do not use labels, define the port and pin constants manually in the `USER CODE BEGIN PD` section.

## Step 4 — Provide Target Firmware Binaries

The firmware binaries (`bootloader.bin`, `partition-table.bin`, `app.bin`) can be supplied to the application in any way that fits your project — loaded from external flash, received over a communication interface, or compiled directly into the firmware image.

One ready-made option is to use the `bin2array.cmake` script included in this repository. It converts every `.bin` file in a directory into C byte arrays (e.g. `bootloader.bin` → `extern const uint8_t bootloader_bin[]`) and generates a single `.c` file to be compiled into your project:

```cmake
include(${FLASHER_DIR}/examples/common/bin2array.cmake)
create_resources(/path/to/binaries
                 ${CMAKE_CURRENT_BINARY_DIR}/target_firmware_data.c)
set_property(SOURCE ${CMAKE_CURRENT_BINARY_DIR}/target_firmware_data.c PROPERTY GENERATED 1)

target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    ${CMAKE_CURRENT_BINARY_DIR}/target_firmware_data.c
)
```

The `extern` symbols generated for each `.bin` file (e.g. `bootloader_bin`, `bootloader_bin_size`) can then be used directly in the application code.

## Step 5 — Build

```bash
cmake -B build -G Ninja
cmake --build build
```

If you generated CubeMX presets:

```bash
cmake --preset Debug
cmake --build --preset Debug
```

For more details on esp-serial-flasher configuration options, see the top-level [README.md](../../README.md).
